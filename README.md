# 📚 E-Learning Platform Data Analysis using MySQL

A SQL-based data analysis project that explores an e-learning platform database to understand learner purchasing behavior, course popularity, and revenue generation. The project demonstrates the use of SQL joins, aggregation functions, and analytical queries to extract actionable business insights.

---

## 📌 Project Overview

Online learning platforms generate valuable transactional data that can be analyzed to improve business performance. This project analyzes an e-learning database using MySQL to identify learner spending patterns, popular courses, revenue by category, and courses with little or no engagement.

The analysis leverages SQL joins and aggregation techniques to answer key business questions and support data-driven decision-making.

---

## 🎯 Objectives

- Analyze learner purchase behavior
- Identify the most popular courses
- Calculate learner spending
- Evaluate category-wise revenue
- Discover learners enrolled across multiple categories
- Identify courses with no purchases
- Demonstrate SQL joins and aggregation techniques

---

## 📂 Database Structure

The project consists of three relational tables:

### Learners

Stores learner information including:

- Learner ID
- Learner Name
- Country

### Courses

Contains course details such as:

- Course ID
- Course Name
- Category
- Price

### Purchases

Stores transaction records linking learners and courses.

- Purchase ID
- Learner ID
- Course ID
- Purchase Date

## 📊 Business Questions Answered

The analysis addresses the following questions:

1. What is the total spending by each learner?
2. Which are the top 3 most purchased courses?
3. What is the revenue generated by each course category?
4. How many unique learners purchased courses in each category?
5. Which learners have purchased courses from multiple categories?
6. Which courses have never been purchased?

---

## 🔍 Key Findings

- SQL joins successfully combined learner, course, and purchase data into meaningful reports.
- Total spending analysis identified high-value learners based on purchase history.
- Top purchased courses highlighted the most popular learning content.
- Category-wise revenue analysis revealed the highest-performing course categories.
- Unique learner counts measured customer engagement across categories.
- Multi-category learners demonstrated diverse learning interests.
- Courses with no purchases were identified, helping highlight low-performing or less popular offerings.
- Aggregation functions simplified complex business reporting and enabled data-driven insights.
